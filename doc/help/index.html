OwlGraph User Manual


Owlgraph belongs to the gargantuan class of graph viewer applications. It differs from most of them in its ability to display “transforms” of arbitrary graphs, and then recalculate those transforms in real-time as the user modifies the original graph.


Owlgraph is a python-gtk project written by Zachary Rubenstein largely at the service of the Rice University VIGRE program. It was starting in Spring 2010.


Opening OwlGraph


Owlgraph is a python script. To start Owlgraph, run owlgraph.py in the program's root directory with python. See README in the program's root directory for installation instructions.


Using the Workspace


When Owlgraph opens, the majority of the window is taken up by a blank workspace. You may clear the workspace at any time with file->New.


The general operation of Owlgraph follows the following schema:

  1. Select the object which you wish to work with on the workspace.

  2. Right click on the workspace to bring up the context menu for that object

  3. Select the operation which you wish to perform.


Objects that can be selected include: nothing, a graph or transform of a graph, a number of graphs and transforms in any combination, a vertex on a graph or transform, and a number of vertices on a graph or transform.


If an object has a filled box in the lower-right hand corner, that object can be resized by clicking and dragging the box. Objects can be moved by clicking and dragging a portion of the object that is not in said filled box. Vertices in transforms cannot be moved or resized independently, but they will reflect changes in the original graph.


Using the Menu


File →

New: Clear the current workspace

Save Workspace: Save the layout of the current workspace to a .ogr file

Save Workspace As: As above, but the .ogr file can be respecified

Open Workspace: Replace the current workspace with one specified in an .ogr file

Export as Picture: Save the current workspace as a .png or .jpg. (The type of picture is judged according the the specified file extension. Owlgraph will choose .png if a valid extension is not chosen manually.)

Quit: Exit Owlgraph

Help →

Owlgraph Help: Open this document.

About Owlgraph: Summon the “about” dialog.


Non-Transform Operations


Each of these transforms is available if the specified object is highlighted or if a child of the specified object is highlighted. (Graphs and transforms are children of “nothing” vertices are children of their respective graphs and transforms.)


Nothing:

New Graph: Creates a New graph at the position of the right click. In the “New Graph” dialog, the user may specify a number of nodes to initialize with this graph, and, if edge probability is set greater than 0, the new graph will be a Bernoulli random graph with the specified edge probability.

Graph and Transforms:

Hide: Hide this graph or transform. When the graph or transform is again selected, with option to unhide the graph will be available. A hidden graph has only a restricted set of operations available.

Add Point: Insert a new vertex into the graph at the position of the right click. This is not available for transforms.

Delete Graph: Remove this graph or transform and all transforms of the graph or transform from the workspace.

[Create Transform]: In addition to these options, a number of transforms are available. These create a transform of the original graph or transform at the position of the right click. The figure depicted in the transform will be the original graph or transform, with edges and vertices selected by some operation specified by the particular transform chosen. For instance, the “Spanning tree” transform will highlight a spanning tree of the transformed graph or transform. Some of these transforms will break if some number of vertices of the graph or transform to be transformed are not selected. These transforms are recalculated when a change is made to the original graph.

Vertices:

Remove Point: Removes the vertex from the graph.

Connect: Mutually connect all selected vertices.

Disconnect: Mutually disconnect all selected vertices.

Toggle active: Highlight nodes that are not highlighted nodes and remove the highlight from highlighted nodes.

(Note: None of these are available for nodes of transforms.)


Creating Transforms


Owlgraph is built to allow addition of custom transforms written in python and using numpy. In order to create a custom transform:

  1. Transforms are located in src/model/adapter/policy/transforms. First, add a new .py file to this directory (let's call it “myTrans.py”)

  2. Copy the contents of identity.py to this new file. This will be the template for the new transform.

  3. Edit the “name =” field to the name of your transform. (For example, change it to:

      name = 'My Transform'

  4. Add to the __init__ method after the first line to set up the transform's initial environment. (the transform will always have access to the field self.selected, which is a list of the indices of the vertices that were selected to create the transform. It should be noted that Owlgraph will attempt to renumber these vertices if a vertex is removed from the original graph.

  5. Alter the transform method to highlight the desired nodes. The user is given three fields:

    1. model: a 2-dimensional numpy array representing the graph's adjacency matrix.

    2. active_nodes: A list of nodes that are already highlighted.

    3. active_edges: a list of edges that are already highlighted. Edges are represented as tuples of vertex indices, arranged in ascending order.

  6. The transform method is expected to return a tuple of two entries:

    1. A list of nodes to be highlighted after the transform.

    2. A list of edges to be highlighted after the transform. (For convenience, you may use self.mk_con(index1, index2) to return a proper edge representation given the indices of two nodes.

    3. Other Convenience methods:

      1. self.path_from_nodes(nodelist) returns a list of edges that represent the path between the nodes given in nodelist in order.

      2. self.all_combinations_from_nodes(nodelist) returns a list of all possible edges between the nodes in nodelist.

  7. Edit __init__.py and add the base name of your file to the list __all__. (For instance, in our case, add the string, 'myTrans' to the list.


Warning: A workspace that employs a given transform must be loaded in a copy of owlgraph with the same transform available.
